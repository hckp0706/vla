# M6：数据语义化转换工具链 技术方案

## 1. 模块定位与功能概述

在整个雷达组网仿真系统中，M6 扮演着**"翻译官"**的角色。M2 融合航迹以高频时序 JSON 输出，包含经纬度、速度、航向、SNR、敌我属性等大量数值字段。直接将原始数据输入大模型（M3），会因上下文过长、数值密集导致大模型无法有效理解态势。

M6 的核心任务是：将 M2 输出的数字航迹压缩为**多段式结构化自然语言战报**，使大模型能高效理解每一批目标的飞行路径、行为特征和威胁模式。

### 核心能力

| 能力 | 说明 |
|------|------|
| M2 融合航迹解析 | 单帧/多帧 JSON 格式自动识别，按 track_id 分组为独立时序数据 |
| 内置地理知识库 | 52 个城市、8 个重点区域、30 个机场坐标，支持坐标→地名反查 |
| 五维航迹特征提取 | 时间/空间/运动/探测质量/身份属性共 5 类特征提取 |
| 航迹分段描述 | 按航向变化 >15° 自动切分飞行段，多段式路径描述 |
| 统计自适应骤变检测 | mean+2σ 阈值 + 绝对值过滤（高度>200m，速度>50km/h） |
| 飞行特征推断 | 11 种基于规则的飞行标签（低空突防/高空侦察/超音速等） |
| 城市途经检测 | 100km 阈值内匹配城市，按时间插入航段描述 |
| 雷达站名称映射 | RADAR_01→青岛雷达站等可读名称 |
| 机场起降识别 | 5km 阈值匹配机场，精确到机场名称 |
| 自然语言战报生成 | 多段式、事件交织的结构化文本输出 |
| 全局态势汇总 | 自动生成目标总数/敌我分布/最近目标/最高威胁汇总 |
| Flask API 集成 | `/api/m6/convert` 供 M4 前端调用，结果自动保存 |
| 命令行 + Python API | `python -m m6_semantic_tool` 和 `SemanticConverter` 双入口 |

## 2. 系统架构与模块划分

### 2.1 模块结构

```
m6_semantic_tool/
├── models.py                  # 数据模型定义（含航迹分段/骤变/城市途经模型）
├── geo_data.py                # 内置地理知识库（城市/区域/机场坐标 + 反查函数）
├── feature_extractor.py       # 航迹特征提取器（五维特征 + 分段 + 骤变 + 途经）
├── flight_inferencer.py       # 飞行特征推断器（11种规则标签）
├── narrative_generator.py     # 自然语言描述生成器（多段式战报 + 雷达名称映射）
├── converter.py               # 核心转换器（组合各模块，支持文件/数据输入）
├── __init__.py                # 包导出（SemanticConverter）
├── __main__.py                # 命令行入口
└── requirements.txt           # 依赖声明
```

### 2.2 模块职责说明

| 模块 | 职责 | 核心功能 |
|------|------|----------|
| **models** | 数据模型 | 定义 FlightSegment/AltitudeAnomaly/SpeedAnomaly/CityOverflight 等分段与异常数据结构；TrackFeatures（五维特征）、TrackDescription、SituationSummary、SituationReport |
| **geo_data** | 地理知识库 | 52个中国城市坐标、8个重点区域经纬度范围、30个机场坐标；Haversine 距离计算；reverse_lookup() 坐标反查（机场5km→城市100km→区域→坐标降级） |
| **feature_extractor** | 特征提取 | 从 FusedTrackPoint 时序数据提取五维特征；航向转折点检测（>15°产生新段）；统计自适应骤变检测（mean+2σ + 绝对值过滤）；城市途经检测（100km）；传感器最近距离（Haversine） |
| **flight_inferencer** | 飞行推断 | 11种规则标签推断：低空飞行/高空飞行/平稳巡航/正常起降/高速飞行/超音速飞行/低速飞行/航线机动/低空突防（FOE+低空）/高空侦察（FOE+高空+高速）/航迹断续 |
| **narrative_generator** | 语言生成 | 多段式飞行路径描述生成（时间区间→方位角→方向→高度变化）；城市途经/骤变事件按时间交织插入；雷达站名称映射；机场级起降描述；中文时间格式（14时21分48秒） |
| **converter** | 核心转换 | 加载 radar_config.json + sensor_config.json 获取传感器坐标和名称；实例化 extractor/inferencer/generator 组合调用；convert()/convert_from_file() 双入口；自动保存到 output/m6_semantic_tool/ |
| **__main__** | CLI 入口 | -i 必选输入文件，-o 可选输出文件，--format text/json 切换格式 |

## 3. 数据模型定义

### 3.1 航迹分段与异常模型

**FlightSegment（飞行段）**：

| 字段 | 类型 | 说明 |
|------|------|------|
| start_time | datetime | 段起始时间 |
| end_time | datetime | 段结束时间 |
| heading_deg | float | 方位角（0-360°） |
| heading_direction | str | 八方位方向（东/南/西/北/东南/东北/西南/西北） |
| start_alt_m | float | 段起始高度（米） |
| end_alt_m | float | 段结束高度（米） |
| avg_alt_m | float | 段平均高度（米） |
| start_speed_kmh | float | 段起始速度（km/h） |
| end_speed_kmh | float | 段结束速度（km/h） |
| avg_speed_kmh | float | 段平均速度（km/h） |
| start_lon / start_lat | float | 段起始经纬度 |
| end_lon / end_lat | float | 段结束经纬度 |

**AltitudeAnomaly（高度骤变）**：

| 字段 | 类型 | 说明 |
|------|------|------|
| time | datetime | 骤变时间 |
| alt_before_m | float | 骤变前高度（米） |
| alt_after_m | float | 骤变后高度（米） |
| change_rate_mps | float | 变化速率（米/秒） |

**SpeedAnomaly（速度骤变）**：

| 字段 | 类型 | 说明 |
|------|------|------|
| time | datetime | 骤变时间 |
| speed_before_kmh | float | 骤变前速度（km/h） |
| speed_after_kmh | float | 骤变后速度（km/h） |
| change_rate_kmhps | float | 变化速率（km/h/s） |

**CityOverflight（城市途经）**：

| 字段 | 类型 | 说明 |
|------|------|------|
| time | datetime | 途经时间 |
| city_name | str | 城市名称 |
| distance_km | float | 距城市中心距离（km） |
| lon / lat | float | 途经点经纬度 |

### 3.2 五维特征模型

**TimeFeatures（时间特征）**：
- first_seen / last_seen：首次/末次出现时间
- duration_seconds：持续时长

**SpatialFeatures（空间特征）**：
- start_lon/lat + start_name：起始位置 + 地理反查名称
- end_lon/lat + end_name：终止位置 + 地理反查名称
- waypoints：航向转折点列表（>30°变化标记）
- sensor_distances：各传感器最近距离及对应时刻
- segments：飞行段列表（FlightSegment[]）
- altitude_anomalies：高度骤变列表
- speed_anomalies：速度骤变列表
- city_overflights：城市途经列表

**MotionFeatures（运动特征）**：
- max_speed_kmh / avg_speed_kmh：最大/平均速度
- heading_direction / heading_deg：主要航向方向
- min_alt_m / max_alt_m / avg_alt_m：高度范围

**QualityFeatures（探测质量特征）**：
- quality_distribution：HIGH/MEDIUM/LOW/LOST 占比
- avg_snr_db：平均信噪比
- gap_count / max_continuous_duration：连续性分析

**IdentityFeatures（身份属性特征）**：
- identity / identity_confidence：敌我属性及置信度
- squawk / callsign / icao24：SSR信息
- emitter_type / emitter_threat_level：辐射源信息
- comm_activity：通信活动

### 3.3 输出模型

| 模型 | 说明 |
|------|------|
| TrackFeatures | 单条航迹的完整特征（包含 flight_labels） |
| TrackDescription | 航迹ID + 自然语言 narrative + 结构化 features |
| SituationSummary | 全局摘要（目标总数/敌友数量/最近目标/最高威胁） |
| SituationReport | 所有 TrackDescription + SituationSummary，提供 to_text()/to_dict() |

## 4. 地理知识库

### 4.1 数据规模

| 类别 | 数量 | 示例 |
|------|------|------|
| 主要城市（MAJOR_CITIES） | 52 个 | 北京、上海、广州、深圳、成都、武汉、沈阳等 |
| 重点区域（KEY_REGIONS） | 8 个 | 渤海海域、黄海海域、东海海域、南海海域、台湾海峡、琼州海峡、朝鲜半岛、日本列岛 |
| 主要机场（MAJOR_AIRPORTS） | 30 个 | 北京首都/ZBAA、上海虹桥/ZSSS、广州白云/ZGGG 等 |

### 4.2 坐标反查逻辑

```
reverse_lookup(lon, lat)
    │
    ├── 优先级1：机场匹配（5km 阈值内） → 返回机场名称
    │
    ├── 优先级2：城市匹配（100km 阈值内） → 返回最近城市名 + "附近"
    │
    ├── 优先级3：区域匹配（经纬度范围判断） → 返回区域名 + "上空"
    │
    └── 降级：无法匹配 → 返回 "坐标({lon},{lat})附近"
```

### 4.3 核心函数

| 函数 | 说明 |
|------|------|
| `haversine_distance(lat1, lon1, lat2, lon2)` | 基于 Haversine 公式的大圆距离计算（km） |
| `reverse_lookup_city(lon, lat)` | 100km 阈值内返回最近城市名 |
| `reverse_lookup_region(lon, lat)` | 判断坐标是否在重点区域经纬度范围内 |
| `reverse_lookup_airport(lon, lat)` | 5km 阈值内返回最近机场名 |
| `reverse_lookup(lon, lat)` | 统合反查（机场→城市→区域→坐标降级） |

## 5. 核心处理逻辑

### 5.1 航迹分段（按航向变化）

```
输入：航迹时序点列表（按时间排序）
  │
  ▼
遍历相邻点，计算 heading 变化量：
  - 处理 0°/360° 环绕（使用 atan2 计算 circular difference）
  - 当变化 > 15° 时触发新段
  ▼
每段输出：
  - 时间区间（"14时00分00秒-14时21分42秒"）
  - 方位角 + 八方位方向（"朝方位角180°向南方向飞行"）
  - 高度变化（"高度从28米爬升至9997米"）
```

### 5.2 统计自适应骤变检测

```
输入：全航迹高度/速度序列
  │
  ▼
1. 计算相邻变化量 Δ[i] = |x[i] - x[i-1]|
2. 计算统计阈值：threshold = mean(Δ) + 2 × std(Δ)
3. 遍历相邻点，当 Δ[i] > threshold 时标记候选：
   - 高度候选：还需满足 Δ[i] > 200m（防小波动误报）
   - 速度候选：还需满足 Δ[i] > 50km/h（防小波动误报）
  ▼
输出：按时间排序的异常事件列表
```

### 5.3 城市途经检测

```
输入：航迹每个时序点、MAJOR_CITIES 坐标表
  │
  ▼
遍历每个时序点：
  - 计算与所有城市的大圆距离
  - 若距离 < 100km，记录途经
  - 同一城市只保留最近的一次途经
  ▼
输出：按时间排序的城市途经列表
```

### 5.4 事件交织插入

```
输入：飞行段列表 + 城市途经列表 + 骤变事件列表
  │
  ▼
将所有事件按时间合并排序：
  for 每个飞行段：
    输出段头（时间区间 + 航向 + 高度变化）
    遍历属于该时段的事件（按时间顺序）：
      if 城市途经：输出 "xx时刻，途经{城市}上空（距离{xx}km）"
      if 高度骤变：输出 "xx时刻高度发生骤变，从{xx}米变化到{xx}米"
      if 速度骤变：输出 "xx时刻速度发生骤变，从{xx}km/h变化到{xx}km/h"
```

### 5.5 飞行特征推断规则

| 推断规则 | 输出标签 | 优先级说明 |
|---------|---------|-----------|
| 敌我属性为 FOE 且平均高度 < 1000m | **低空突防** | 替换"低空飞行" |
| 敌我属性为 FOE 且高空飞行 + 高速飞行 | **高空侦察** | 替换"高空飞行" |
| 平均高度 < 1000m | 低空飞行 | — |
| 平均高度 > 12000m | 高空飞行 | — |
| 高度变化范围 < 500m | 平稳巡航 | — |
| 存在明显爬升段和下降段 | 正常起降 | — |
| 最大速度 > 1500km/h | 超音速飞行 | — |
| 最大速度 > 900km/h | 高速飞行 | — |
| 平均速度 < 300km/h | 低速飞行 | — |
| 航向变化含 ≥2 个转折点 | 航线机动 | — |
| 存在探测间隙 (gap_count > 0) | 航迹断续 | — |

## 7. 接口规范

### 7.1 命令行接口

```bash
python -m m6_semantic_tool -i <M2综合融合航迹文件> [-o <输出文件>] [--format text/json]
```

| 参数 | 必选 | 说明 |
|------|------|------|
| `-i / --input` | 是 | M2 输出的综合融合航迹 JSON 文件路径 |
| `-o / --output` | 否 | 输出文件路径（默认输出到控制台） |
| `--format` | 否 | 输出格式：text（默认）或 json |

**输入文件要求**：
- 支持单帧格式（一个对象，含 `network_tracks` 数组）
- 支持多帧格式（数组，每个元素含 `network_tracks` 数组）
- 支持 `fused_comprehensive.json`（含敌我识别/SSR/辐射源信息）

### 7.2 Python API

```python
from m6_semantic_tool import SemanticConverter

converter = SemanticConverter(
    radar_config_path="m2_sensor_simulation/radar_config.json",
    sensor_config_path="m2_sensor_simulation/sensor_config.json"
)

report = converter.convert_from_file("output/m2_tracks/comprehensive_.../fused_comprehensive.json")

print(report.to_text())     # 完整自然语言文本
print(report.to_dict())     # 结构化字典
print(report.summary)       # SituationSummary 对象
```

### 7.3 Flask API（供 M4 调用）

| 端点 | 方法 | 请求 | 响应 |
|------|------|------|------|
| `/api/m6/convert` | POST | `{"network_tracks": [...], "track_id": "TRK_001"}` | `{"success": true, "text": "...", "summary": "...", "saved_json": "...", "saved_txt": "..."}` |
| `/api/m6/save` | POST | `{"text": "...", "summary": "..."}` | `{"success": true, "saved_txt": "..."}` |

## 10. 技术亮点

1. **内置地理知识库**：无需外部 GIS 服务，52 城市+8 区域+30 机场坐标硬编码，毫秒级反查
2. **多段式航迹描述**：按航向变化自动分段，每段独立描述方位角、方向、高度变化，大模型更易理解
3. **统计自适应异常检测**：mean+2σ 阈值随航迹自适应 + 绝对值下限过滤，兼顾灵敏度与误报率
4. **事件交织插入**：城市途经、高度骤变、速度骤变按时间正序交织，形成完整的时间线叙事
5. **雷达站名称映射**：radar_config.json → 自然语言，RADAR_01→青岛雷达站，提高可读性
6. **机场级识别**：5km 阈值匹配机场 + 100km 城市 + 区域 + 坐标四级降级链
7. **中文时间格式**：14时21分48秒，自然流畅
8. **M4-Flask 深度集成**：单文件启动，前后端一体化；转化结果自动保存，支持 JSON + TXT 双格式
9. **零网络依赖**：纯 Python 代码工具，无大模型 API、无 GIS 服务调用
10. **灵活接入 M3**：`to_text()` 返回值可直接拼接进大模型 prompt
