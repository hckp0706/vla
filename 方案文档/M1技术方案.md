# M1：意图驱动航迹生成器 技术方案

## 1. 模块定位与功能概述

在整个雷达组网仿真系统中，M1 扮演着**"战场导演"**的角色。它的核心任务是：接收指挥员下达的飞行任务指令（飞行简令），结合真实的地理环境、空域航路规则和飞机性能参数，推演出一条符合物理规律与航空逻辑的1Hz高精度飞行轨迹。

M1 生成的数据是整个系统数据流的源头。它不关心雷达能不能看到目标，它只负责如实地模拟出飞机在每一秒（1Hz频率）的位置、速度、姿态以及电磁反射特征（RCS），并将这些数据打包向后传递给 M2（雷达仿真引擎）。

### 核心能力

| 能力 | 说明 |
|------|------|
| 飞行简令解析 | 使用正则表达式解析半结构化飞行简令文本，支持新旧4种格式 |
| 三级降级航路规划 | EAIP官方数据 → GLM-5.1大模型 → 本地预设航路 |
| EAIP航路规划 | BFS算法在真实航路网络中查找最优路径 |
| 大圆航线计算 | WGS84椭球模型计算地球表面最短路径 |
| 平滑转弯算法 | 基于标准转弯率（3度/秒）实现圆弧平滑过渡 |
| 高度剖面计算 | 模拟起飞→爬升→巡航→下降→降落全过程 |
| 跑道方向应用 | 起飞/降落阶段使用真实跑道方向作为航向 |
| 动态RCS估算 | 根据飞行姿态（机头/侧面/起落架）和任务类型计算 |
| 地形跟随模式 | 低空突防、对地攻击等任务支持贴地飞行 |

## 4. 飞行简令解析

### 4.1 支持的简令格式

**新格式1（带途径点）**：

**新格式2（简化）**：
```
[批号] [机型] [任务类型] 于 [时间] 从 [起点] 起飞，[降落动作] [降落地点]。
```

**旧格式1（带途径点，兼容）**：
```
[批号] [机型] 于 [时间] 从 [起点] 起飞，执行[任务类型]任务，[途径动作] [途径地点] [途径时间]，[降落动作] [降落地点] [降落时间]。
```

**旧格式2（简化，兼容）**：
```
[批号] [机型] 于 [时间] 从 [起点] 起飞，执行[任务类型]任务，[降落动作] [降落地点]。
```

```
简令文本

    └── 验证
            ├── 必填字段检查（批号、机型、起飞时间、起飞地点）
```

### 6.2 EAIP航路规划算法

**算法1：BFS航路网络搜索**

在加载的航路网络拓扑中，使用BFS搜索起终点之间的最优路径：

```python
def _find_route_in_network(self, start_wp, end_wp):
    """
    BFS搜索航路网络中的最优路径
    
    约束条件：
    - 路径最大长度20个节点
    - 相邻航路点距离不超过10度
    - 总偏移不超过起终点距离+5度
    """
    queue = deque([(start_wp, [start_wp])])
    visited = {start_wp: 0}
    
    while queue:
        current, path = queue.popleft()
        if current == end_wp:
            return path
        if len(path) >= 20:
            continue
        for neighbor in self.route_network[current]['connections']:
            # 距离和方向约束过滤
            if self._is_valid_neighbor(neighbor, path, start_wp, end_wp):
                visited[neighbor] = len(path)
                queue.append((neighbor, path + [neighbor]))
    return None
```

### 6.3 降级机制

| 场景 | 数据源 | 优先级 | 触发条件 |
|------|--------|--------|----------|

### 6.4 大模型航路点生成

**LLMClient** 封装了GLM-5.1 API调用，支持两种提示词模板：

**军机模板**：强调军事任务特征，提示大模型生成战术航线

**民航模板**：强调民航航线特征，提示大模型生成标准民航航路点

```python
class LLMClient:
    def generate_prompt(self, intent_dict):
        """根据飞行意图生成提示词（区分军机/民航）"""
        is_military = self._is_military_aircraft(intent_dict['platform_type'])
        if is_military:
            return self._build_military_prompt(intent_dict)
        else:
            return self._build_civil_prompt(intent_dict)
    
    def generate_route(self, prompt):
        """调用大模型API生成航路点JSON"""
        response = requests.post(
            self.api_url,
            headers={"Authorization": f"Bearer {self.api_key}"},
            json={
                "model": self.model,
                "messages": [{"role": "user", "content": prompt}],
                "temperature": self.temperature,
                "max_tokens": self.max_tokens
            },
            timeout=60
        )
        return self._parse_llm_response(response)
```

## 7. 核心算法详解

### 7.1 大圆航线计算与插值

使用 `geographiclib` 库的WGS84椭球模型进行精确地理计算：

**距离计算**：

```python
def _geodesic_distance(self, lat1, lon1, lat2, lon2):
    """使用WGS84椭球模型计算两点间大圆距离（米）"""
    result = self.geod.Inverse(lat1, lon1, lat2, lon2)
    return result['s12']
```

**航线插值**：

```python
def _geodesic_interpolate(self, lat1, lon1, lat2, lon2, fraction):
    """
    在大圆航线上进行插值
    
    参数：
        fraction: 插值比例（0=起点，1=终点）
    
    返回：
        (插值点纬度, 插值点经度, 起点方位角)
    """
    result = self.geod.Inverse(lat1, lon1, lat2, lon2)
    total_distance = result['s12']
    azimuth = result['azi1']
    interp_distance = total_distance * fraction
    interp_result = self.geod.Direct(lat1, lon1, azimuth, interp_distance)
    return interp_result['lat2'], interp_result['lon2'], azimuth
```

**航向计算**：

```python
def _calculate_heading(self, lat1, lon1, lat2, lon2):
    """计算两点之间的航向角（0-360度，真北为0）"""
    result = self.geod.Inverse(lat1, lon1, lat2, lon2)
    heading = result['azi1']
    if heading < 0:
        heading += 360
    return heading
```

### 7.2 平滑转弯算法

基于标准转弯率（3度/秒）计算转弯半径，判断飞机是否处于转弯区域：

```python
# 转弯半径 = 速度 / 转弯角速度
turn_radius = speed_ms / math.radians(TURN_RATE_DEG_PER_SEC)

def _is_in_turn(self, waypoints, segment_distances, current_distance, turn_radius):
    """
    判断当前位置是否在转弯区域内
    
    转弯区域定义：航路点前后各一个转弯半径的范围内
    """
    accumulated_distance = 0.0
    for i in range(len(segment_distances)):
        seg_end = accumulated_distance + segment_distances[i]
        turn_start = seg_end - turn_radius
        turn_end = seg_end + turn_radius
        if turn_start <= current_distance <= turn_end and i < len(segment_distances) - 1:
            return True
        accumulated_distance = seg_end
    return False
```

### 7.3 高度剖面生成

高度剖面采用5阶段模型，基于距离而非时间计算高度变化：

```
高度
  │
  │                    ┌──────────────┐  巡航
  │                   ╱                ╲
  │                  ╱                  ╲
  │                 ╱                    ╲
  │    起飞→爬升───╱                      ╲───下降→降落
  │
  └─────────────────────────────────────────── 距离
       │ climb_end│                    │descent_start
```

**计算逻辑**：

```python
def _calculate_altitude_profile(self, current_distance, total_distance, 
                                start_alt, cruise_alt, end_alt,
                                climb_distance, descent_distance, aircraft):
    """
    高度剖面计算
    
    阶段判定：
    - GROUND_TAKEOFF: current_distance <= climb_end 且 爬升进度 < 10%
    - CLIMBING: current_distance <= climb_end 且 爬升进度 >= 10%
    - CRUISING: climb_end < current_distance < descent_start
    - DESCENDING: current_distance >= descent_start 且 下降进度 < 90%
    - LANDING: current_distance >= descent_start 且 下降进度 >= 90%
    """
    climb_end = climb_distance
    descent_start = total_distance - descent_distance
    
    if current_distance <= climb_end:
        progress = current_distance / climb_distance
        alt = start_alt + (cruise_alt - start_alt) * progress
        vertical_rate = aircraft.climb_rate_ms
        phase = GROUND_TAKEOFF if progress < 0.1 else CLIMBING
    elif current_distance >= descent_start:
        progress = (current_distance - descent_start) / descent_distance
        alt = cruise_alt - (cruise_alt - end_alt) * progress
        vertical_rate = -aircraft.descent_rate_ms
        phase = LANDING if progress > 0.9 else DESCENDING
    else:
        alt = cruise_alt
        vertical_rate = 0.0
        phase = CRUISING
    
    return alt, vertical_rate, phase
```

**爬升/下降距离计算**：

```python
climb_distance = (cruise_alt - start_alt) / aircraft.climb_rate_ms * speed_ms
descent_distance = (cruise_alt - end_alt) / aircraft.descent_rate_ms * speed_ms
```

**任务高度调整**：

```python
def _get_mission_altitude(self, aircraft, mission):
    """取任务典型高度和机型巡航高度×高度因子的较小值"""
    if mission:
        return min(mission.typical_altitude_m, aircraft.cruise_alt_m * mission.altitude_factor)
    return aircraft.cruise_alt_m
```

**地形跟随模式**：当地形跟随任务（低空突防、反舰、对地攻击）巡航时，高度限制为任务巡航高度。

### 7.5 RCS估算逻辑

RCS的动态计算遵循4条规则：

```python
def _calculate_rcs(self, aircraft, phase, is_turning, heading,
                   waypoints, segment_distances, current_distance, rcs_factor):
    """
    RCS估算规则：
    1. 基准值：机头方向RCS（nose_rcs_dbsm）
    2. 起降增量：起飞/降落阶段起落架放下，RCS增加gear_rcs_increment
    3. 转弯增量：转弯时侧面暴露，RCS从机头值向侧面值线性插值
    4. 任务因子：根据任务类型的rcs_factor调整
    """
    base_rcs = aircraft.nose_rcs_dbsm
    
    # 规则2：起降增量
    if phase in [GROUND_TAKEOFF, LANDING]:
        base_rcs += aircraft.gear_rcs_increment
    
    # 规则3：转弯增量（侧面暴露插值）
    if is_turning:
        turn_progress = self._get_turn_progress(waypoints, segment_distances, current_distance)
        side_rcs = aircraft.side_rcs_dbsm
        base_rcs = base_rcs + (side_rcs - base_rcs) * turn_progress
    
    # 规则4：任务因子
    base_rcs = base_rcs * rcs_factor
    
    return base_rcs
```

**典型机型RCS值**：

| 机型 | 机头RCS (dBsm) | 侧面RCS (dBsm) | 起落架增量 (dBsm) |

| 民航客机 | 5.0 | 15.0 | 8.0 |
| 歼-20 | -10.0 | 0.0 | 5.0 |
| F-22 | -15.0 | -5.0 | 5.0 |
| B-2 | -20.0 | -10.0 | 5.0 |
| 空警-2000 | 20.0 | 40.0 | 20.0 |

- **大圆航线邻近筛选**：先粗筛（1.5度阈值），再精排，减少不必要的精确计算

- **优先使用本地数据**：EAIP数据无需网络请求，响应最快
- **大模型结果临时缓存**：验证通过的航路点临时添加到知识库，避免重复查询

## 14. 技术亮点

1. **EAIP官方数据优先**：整合184个机场、1418个航路点的真实航空数据，确保航线规划的真实性
2. **三级降级机制**：EAIP→大模型→本地预设，保证系统在任何情况下都能生成轨迹
3. **BFS航路网络搜索**：在真实航路网络拓扑中搜索最优路径，航线符合实际航路结构
4. **精确地理计算**：使用geographiclib库的WGS84椭球模型进行大圆航线计算
5. **跑道方向应用**：起飞/降落阶段使用真实跑道方向，提高仿真真实性
6. **平滑转弯算法**：基于标准转弯率消除航向突变，符合真实飞行规律
7. **动态RCS估算**：4规则RCS计算（基准+起落架+转弯插值+任务因子）
8. **军机/民航双模板**：大模型提示词区分军机/民航，生成不同风格的航线
9. **5重航路点验证**：确保大模型返回的航路点数据有效可用
10. **地形跟随模式**：低空突防等任务支持贴地飞行
