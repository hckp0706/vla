# 基于大模型的雷达组网仿真与智能辅助决策系统
## （工程开发任务说明文档 V1.2）

> **文档定位**：本文档为项目启动阶段的顶层设计说明，供后续工程团队阅读、评估工作量与拆解开发任务使用。
> **当前状态**：概念设计与架构规划阶段。
> **核心理念**：参考自动驾驶 VLA（Vision-Language-Action）范式，站在战区联合防空指挥所视角，构建以雷达组网操控为核心的 **“全域感知-认知研判-动作执行”** 端到端闭环系统。

## 1. 项目愿景与核心理念

### 1.1 雷达领域的“VLA”范式映射（指挥决策层视角）
在自动驾驶中，VLA 模型处理摄像头画面与雷达点云，理解路况语义，最终输出方向盘转角。而在现代防空指挥体系中，指挥员面对的是更广阔、更复杂、更具欺骗性的战场空间。本项目将这一范式平移至防空领域，构建如下宏观闭环：

*   **V（Vision - 全域态势感知层）**：不同于自动驾驶单一的视觉传感器，战区感知是多源异构的叠加。
    *   **雷达感知**：我方组网雷达获取的电磁回波，形成的数字点迹与航迹。
    *   **物理环境感知**：影响电磁波传播与飞行器的物理场，如地形遮蔽（高程数据）、气象条件（大气波导、风雨衰减）、海面杂波等。
    *   **情报信息感知**：非实时的定性或半定量信息，如电子战(ESM)测向报文、天基卫星过境侦察图像、开源新闻(OSINT)、上级敌情通报等。
*   **L（Language - 指挥认知研判层）**：相当于自动驾驶的“认知大模型”。其核心不是做数值计算，而是作为“参谋长”，将上述三类感知获取的生硬数字与零散文本，结合指挥经验（知识库），进行**交叉比对、去伪存真、意图推断**，最终输出结构化的战术研判结论。
*   **A（Action - 联合作战动作层）**：对应认知研判后的体系反制。包括宏观的战机起飞、导弹阵地戒备，而**本项目深度聚焦的核心动作是：精细化、智能化的雷达组网操控**（如频率捷变、波束调度、辐射管控、多站协同）。

### 1.2 核心建设目标
1.  **意图驱动仿真**：支持通过自然语言描述目标飞行意图，结合目标机型知识库，地理环境知识库等知识，代码自动生成符合物理规律与意图逻辑的目标真实航迹数据
2.  **高保真物理底座**：真实模拟雷达方程、RCS 起伏、地形遮挡与复杂电磁环境，为认知层提供带有“战场迷雾”的可靠数据源。
3.  **强推理威胁评估**：利用高参数量在线大模型的长上下文与逻辑推理能力，融合多源情报，输出高置信度、可解释的分级威胁评估。
4.  **动态博弈对抗**：构建具备欺骗能力的红方引擎，逼迫蓝方认知模型在不确定与不对称信息下进行动态研判。
5.  **聚焦雷达操控代理**：通过语义桥接机制，让大模型绕开底层物理计算，直接输出可执行的精细化雷达抗干扰与调度指令。

## 2. 系统核心模块详述（按数据流顺序）

整个系统去除了底层的控制调度模块，聚焦业务逻辑。以下按数据流动的先后顺序，详细介绍 9 个核心模块的职责、输入输出及**本质属性**。

### 2.1 感知与生成阶段（V层支撑）

**1. M1：意图驱动航迹生成器**

*   **本质属性**：`大模型代码解释器` + `纯物理动力学代码`
*   **模块作用**：作为战场的“造物主”，将人类指挥员或红方引擎的自然语言意图，转化为符合空气动力学的 1Hz 高精度飞行轨迹。
*   **输入**：半结构化飞行简令（如“0001 歼击机 于...从...起飞，低空突防...”）、机型性能与电磁特征参数库、航路与导航点数据库、地理高程数据（DEM）。
*   **输出**：1Hz 真实航迹序列 JSON（包含：时间戳、真实经纬度、高度、速度、垂直速率、平滑连续的航向、基于姿态动态估算的RCS、飞行阶段枚举如“巡航/转弯”）。

**2. M2：高保真传感器仿真引擎**

* **本质属性**：`纯物理与数学计算代码`（无大模型参与）

* **模块作用**：制造“战场迷雾”。采用功能级等效仿真策略（不仿真底层IQ信号与FFT，直接模拟MTD/CFAR的因果结果），严格遵守雷达视距方程、波束物理扫描遮挡与多普勒杂波抑制逻辑，阻断决策层看到上帝视角。对穿透迷雾的目标注入量测噪声，并进行卡尔曼滤波平滑，输出多站组网融合后的稳定航迹。

* **输入**：M1 的真实航迹序列、多站雷达组网阵地坐标与物理参数（如架高、扫描周期、波束宽）、当前雷达工作模式（受 M7 控制）。

* **输出**：**组网融合航迹 JSON 列表**。未被发现的目标直接丢弃，不生成记录。输出的是经过滤波平滑的稳定坐标，并附带探测质量与多源信息。*航迹 JSON 示例*：

  ```json
  {
    "track_id": "T01", 
    "time": "10:05:01", 
    "source_radars": ["RADAR_01", "RADAR_04"],
    "obs_lon": 118.5215, 
    "obs_lat": 24.1095, 
    "obs_alt": 485.0,
    "obs_speed_ms": 245.0,
    "track_quality": "HIGH", 
    "snr_avg_db": 15.2
  }
  ```

**3. M8：红方博弈对抗引擎**

*   **本质属性**：`独立智能体`（包含规则状态机 + 独立大模型会话）
*   **模块作用**：扮演“假想敌”，提供动态对抗压力。根据蓝方雷达当前暴露的状态，动态调整红方战术，并生成虚假/真实的多源情报。
*   **输入**：蓝方雷达部分暴露参数（模拟被红方侦察到）、初始红方剧本、全局时钟。
*   **输出**：红方战术调整指令（注入 M1 修改航迹）、生成的多源文本情报（如“*开源新闻：某民航偏航*”或“*技侦通报：发现 L 波段雷达信号*”，注入 M3）。

### 2.2 认知与决策阶段（L层支撑）

**4. M6：数据语义化转换工具链**
*   **本质属性**：`纯代码工具`（Python 数据处理脚本）
*   **模块作用**：大模型对数字极度不敏感，此模块是“翻译官”，将生硬的数字坐标与物理量转译为大模型擅长理解的自然语言战报。
*   **输入**：M2 输出的点迹 JSON 列表（包含 snr, rcs 等数字）、雷达阵地坐标。
*   **输出**：结构化的自然语言态势文本。例如：“*目标 T01 在过去 10 秒内断续出现 3 次，信噪比极低（约10dB），估计 RCS 较小（-5dBsm），当前位于 01 号雷达东南方 85 公里处，正在低空逼近。*”

**5. M5：领域知识库与规则引擎**
*   **本质属性**：`工具/检索库`（向量数据库 + 传统数据库）
*   **模块作用**：为认知大模型提供专业的“窗外知识”，弥补其在军事战术领域的知识盲区。
*   **输入**：专家录入的 PDF/Word 手册（机型库、雷达库、电磁对抗战法）、结构化的威胁评判矩阵。
*   **输出**：根据当前态势检索出的 Top-K 相关知识文本片段（供 M3 的 Prompt 调用）。

**6. M3：智能认知与威胁评估中枢**

*   **本质属性**：`大语言模型 Agent`（系统的核心大脑，具备 Tool Calling 能力）
*   **模块作用**：替代人类指挥员进行思考。聚合语义态势、外部情报与检索知识，进行交叉验证，剥离欺骗，推断真实意图，并输出战术动作建议。
*   **输入**：M6 的态势文本、M8 的外部情报文本、M5 检索的知识片段、上一周期的决策上下文。
*   **输出**：结构化 JSON 决策结果（包含：目标 ID、四级威胁等级标签、详细的可解释推理链文本、具体的雷达操控动作意图描述）。

### 2.3 执行与交互阶段（A层支撑）

**7. M7：雷达指令解析与执行器**
*   **本质属性**：`纯代码工具`（参数映射与校验脚本）
*   **模块作用**：作为“传令兵”，将大模型输出的宏观战术意图，精确映射为底层雷达仿真引擎能识别的微观物理参数，并进行安全拦截。
*   **输入**：M3 输出的雷达操控动作意图 JSON。
*   **输出**：下发至 M2 的雷达控制指令 JSON（包含具体的工作频段、脉冲重频、波束驻留时间、波束指向角等参数配置）。

**8. M4：多维态势可视化界面**
*   **本质属性**：`前端交互代码`（Web/桌面 GUI）
*   **模块作用**：人机共生接口。呈现 GIS 态势，允许人类指挥员监督大模型决策并在必要时介入接管。
*   **输入**：M1 真实轨迹（可选上帝视角）、M2 观测点迹、M3 威胁评估与推理链 JSON。
*   **输出**：屏幕渲染画面（威胁高亮、推理链悬浮窗）；人类指挥员的交互指令（覆盖或确认大模型决策，转发给 M7）。

**9. M10：对抗评估与复盘模块**
*   **本质属性**：`规则代码` + `大模型文本生成`
*   **模块作用**：对单次推演进行量化打分，并自动生成包含关键决策节点的复盘分析报告。
*   **输入**：全周期流转的轨迹、点迹、决策日志。
*   **输出**：防御得分、预警时间统计、复盘 Markdown 报告（含大模型生成的战法得失分析）。

## 3. 项目实施路线图（三步走战略）

为降低工程风险，确保系统从概念到实战的稳步演进，项目分为三个阶段逐步推进：

### 第一阶段：系统闭环与逻辑验证（当前阶段重点）
*   **阶段目标**：跑通“意图生成 -> 物理探测 -> 语义转换 -> 大模型评估 -> 指令下发 -> 物理反馈”的完整 VLA 闭环，验证业务逻辑可行性。
*   **技术策略**：
    *   **大模型选型**：全面调用高参数量在线大模型 API（如 GPT-4o、Claude-3.5 等），利用其强大的零样本推理能力和长上下文窗口，快速验证 Prompt 工程与 Tool Calling 路由的有效性。
    *   **物理仿真**：集中精力死磕 M1 和 M2 的物理保真度，确保雷达方程、RCS 起伏、地形遮挡的计算绝对正确。
    *   **对抗设计**：M8 红方引擎采用简单的基于规则的状态机，主要验证情报注入对大模型研判的干扰效果。
*   **交付标准**：能在单机上演示完整的“低空突防+远距支援干扰”对抗剧本，大模型能稳定输出正确的雷达抗干扰建议，并在界面上看到参数改变后探测效果的物理反馈。

### 第二阶段：能力优化与数据沉淀（未来规划）
*   **阶段目标**：解决单轮对话在长时序推演中的“遗忘”问题，深化博弈对抗的复杂度，并开始为模型微调积累高质量数据。
*   **技术策略**：
    *   **长程记忆机制**：在 M3 认知中枢引入摘要记忆与图数据库，使大模型能够结合 30 分钟前的历史态势进行研判。
    *   **强化博弈引擎**：M8 红方引擎升级为具备规划能力的 Agent，能够根据蓝方历史决策习惯，主动实施战术欺骗（如伪装民航、释放诱饵机）。
    *   **数据闭环采集**：完整记录第一、二阶段中，大模型在复杂对抗场景下产生的**“优质推理链”与“正确动作映射”**，构建雷达操控领域的专业指令微调数据集。
*   **交付标准**：系统能支撑 20 个以上目标、长达 1 小时的连续对抗推演；形成包含万级高质量样本的领域数据集。

### 第三阶段：模型下沉与专业微调 SFT（未来远景）
*   **阶段目标**：摆脱对外部在线大模型 API 的依赖，实现低延迟、纯离线、数据保密的本地化部署，获得超越通用大模型的“雷达控制领域专家模型”。
*   **技术策略**：
    *   **模型微调**：利用第二阶段积累的数据集，选取开源轻量级基座模型（如 Qwen2.5-7B/14B 或 Llama-3-8B），进行监督微调（SFT）与强化学习（RLHF/DPO）。
    *   **能力迁移**：将原本依赖复杂 Prompt 和外部工具链才能完成的“空间语义转换”和“规则检索”能力，尽可能内化到小模型的权重中。
*   **交付标准**：在单张消费级显卡（如 RTX 4090）上流畅运行全系统；本地 7B/14B 模型在威胁评估准确率和指令输出合规性上，达到甚至超越第一阶段在线大模型的水平。
