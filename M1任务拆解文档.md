# M1 航迹生成功能 - 任务拆解文档

## 1. 任务概述

根据M1技术方案，需要实现基于EAIP官方数据的航迹生成功能，并支持大模型作为辅助手段。本文档将该功能拆解为具体的开发任务，便于团队协作和进度跟踪。

## 2. 功能需求分析

### 2.1 核心需求

| 需求编号 | 需求描述 | 来源（技术方案章节） |
|----------|----------|---------------------|
| REQ-001 | 优先使用EAIP官方数据生成航路点 | 3.1 数据来源 |
| REQ-002 | 整合EAIP机场数据（184个机场） | 3.3 机场数据覆盖 |
| REQ-003 | 整合EAIP标准航路点数据（1418个） | 3.4 航路点数据覆盖 |
| REQ-004 | 实现EAIP数据加载模块 | 3.5 数据加载模块 |
| REQ-005 | 实现基于EAIP数据的航线规划算法 | 4.2 航路选择算法 |
| REQ-006 | 支持大模型作为EAIP数据的补充 | 4.3 协作模式 |
| REQ-007 | 本地代码负责时间计算（途经点时间、降落时间） | 6.3 时间计算逻辑 |
| REQ-008 | 本地代码负责速度规划和高度剖面生成 | 6.3 速度规划逻辑 |
| REQ-009 | 实现降级机制（EAIP→大模型→本地预设） | 6.4 降级处理机制 |
| REQ-010 | 支持配置EAIP数据路径和大模型参数 | 9.3 配置选项 |

### 2.2 输入输出需求

| 类型 | 格式 | 说明 |
|------|------|------|
| 输入 | 飞行简令 | `[批号] [机型] [任务类型] 于 [起飞时间] 从 [起点] 起飞，途径 [途经点]，降落 [终点]` |
| EAIP输入 | JSON文件 | 机场数据和航路点数据 |
| 大模型输入 | 提示词 | 包含任务描述、机型、地点等信息的结构化提示词 |
| 大模型输出 | JSON | 包含航路点序列（名称、经度、纬度、建议高度） |
| 最终输出 | JSON | 包含每秒轨迹点的完整轨迹文件 |

### 3.2 详细任务说明

#### TASK-001: EAIP数据加载模块

**任务描述**：实现EAIP数据的加载和管理，提供统一的数据访问接口

**需求来源**：REQ-004

**实现内容**：
- 创建EAIP数据加载器类
- 实现机场数据加载方法
- 实现航路点数据加载方法
- 构建空间索引，支持快速查询
- 提供机场和航路点查询接口

**验收标准**：
- 能够成功加载EAIP机场数据（184个）
- 能够成功加载EAIP航路点数据（1418个）
- 支持根据ICAO代码查询机场信息
- 支持根据名称查询航路点信息
- 查询响应时间小于100ms

#### TASK-002: EAIP机场数据整合

**任务描述**：整理和补充EAIP机场数据，确保数据完整性

**需求来源**：REQ-002

**实现内容**：
- 整理GEN_2.4中的机场列表（184个）
- 补充机场精确坐标（已有57个，需要扩展）
- 补充机场标高和跑道信息
- 验证数据准确性

**验收标准**：
- 机场数据完整，184个机场都有基本信息
- 至少80个机场有精确坐标
- 数据格式符合JSON规范
- 数据文件存放在指定位置

#### TASK-003: EAIP航路点数据整合

**任务描述**：整理ENR_4.4中的标准航路点数据

**需求来源**：REQ-003

**实现内容**：
- 解析ENR_4.4 PDF文件
- 提取1418个标准航路点
- 整理航路点坐标和所属航路
- 验证数据准确性

**验收标准**：
- 航路点数据完整，1418个航路点都有坐标
- 每个航路点包含名称、纬度、经度、所属航路
- 数据格式符合JSON规范
- 数据文件存放在指定位置

#### TASK-004: 基于EAIP的航线规划算法

**任务描述**：实现使用EAIP数据进行航线规划的核心算法

**需求来源**：REQ-001, REQ-005

**实现内容**：
- 实现起终点坐标获取
- 实现航路点筛选算法（根据距离和航路归属）
- 实现最优路径选择算法（A*或动态规划）
- 构建完整的航路点序列

**验收标准**：
- 能够根据起终点自动规划合理的航线
- 航线包含合理数量的航路点（5-10个）
- 航路点之间距离合理（150-200公里）
- 支持带途经点的航线规划

#### TASK-005: 时间计算模块

**任务描述**：根据航路点和机型性能参数计算途经点时间和降落时间

**需求来源**：REQ-007

**实现内容**：
- 实现大圆航线距离计算
- 根据飞行阶段计算各航段时间
- 累加计算途经点到达时间
- 计算总飞行时间和降落时间

**验收标准**：
- 时间计算准确，符合机型性能参数
- 途经点时间和降落时间自动生成
- 支持不同机型和任务类型的时间计算
- 飞行时间误差不超过实际时间的5%

#### TASK-006: 速度规划模块

**任务描述**：根据机型类型、任务类型和飞行阶段规划速度剖面

**需求来源**：REQ-008

**实现内容**：
- 定义不同机型的速度参数
- 根据飞行阶段调整速度
- 支持民航和军机的不同速度规划

**验收标准**：
- 速度规划符合机型性能
- 正确处理不同飞行阶段的速度变化
- 支持多种机型和任务类型
- 速度值在合理范围内

#### TASK-007: 高度剖面生成模块

**任务描述**：根据机型爬升/下降率生成平滑的高度变化曲线

**需求来源**：REQ-008

**实现内容**：
- 实现爬升阶段高度计算
- 实现巡航阶段高度保持
- 实现下降阶段高度计算
- 生成平滑的高度变化曲线

**验收标准**：
- 高度剖面符合机型性能参数
- 高度变化平滑，无突变
- 支持不同机型的爬升/下降率
- 最终高度与机场标高匹配

#### TASK-008: 物理约束验证模块

**任务描述**：验证轨迹是否符合物理规律

**需求来源**：REQ-008

**实现内容**：
- 速度约束验证（不超过最大速度）
- 转弯半径约束验证
- 爬升/下降率约束验证
- 高度层约束验证

**验收标准**：
- 能够识别违反物理约束的轨迹
- 提供约束违反的详细信息
- 能够自动调整轨迹以满足约束
- 调整后的轨迹仍符合飞行逻辑

#### TASK-009: 大模型API封装模块

**任务描述**：封装GLM-5.1大模型的API调用，提供统一的接口供其他模块使用

**需求来源**：REQ-006

**实现内容**：
- 创建大模型客户端类
- 实现API调用方法
- 处理API响应和错误
- 支持配置API Key和模型名称

**验收标准**：
- 能够成功调用GLM-5.1 API
- 正确处理API错误和超时
- 支持配置不同的模型参数
- 响应时间符合预期

#### TASK-010: 提示词生成模块

**任务描述**：根据飞行简令生成符合规范的大模型提示词

**需求来源**：REQ-006

**实现内容**：
- 实现提示词模板引擎
- 支持民航和军机两种提示词模板
- 确保提示词中不包含时间信息
- 生成结构化的提示词

**验收标准**：
- 生成的提示词符合技术方案中的格式要求
- 正确填充所有必要参数
- 不包含任何时间相关信息
- 提示词能够引导大模型生成有效航路点

#### TASK-011: 航路点解析与验证模块

**任务描述**：解析大模型返回的航路点，并验证其有效性

**需求来源**：REQ-006

**实现内容**：
- 解析JSON格式的航路点数据
- 验证坐标范围有效性
- 验证航路点数量合理性
- 验证相邻航路点距离合理性

**验收标准**：
- 正确解析大模型返回的JSON数据
- 能够识别无效的航路点数据
- 提供详细的验证错误信息
- 验证逻辑与EAIP数据验证一致

#### TASK-012: 降级机制实现

**任务描述**：实现EAIP→大模型→本地预设的三级降级机制

**需求来源**：REQ-009

**实现内容**：
- 检测EAIP数据是否可用
- 检测大模型调用失败
- 自动切换到下一级数据源
- 记录降级日志
- 保留原始数据便于复核

**验收标准**：
- EAIP有数据时优先使用EAIP
- EAIP无数据时自动切换到大模型
- 大模型失败时自动降级到本地预设航路
- 降级过程对用户透明
- 记录降级日志便于排查

#### TASK-013: 配置管理模块

**任务描述**：实现EAIP数据和大模型相关配置的管理

**需求来源**：REQ-010

**实现内容**：
- 配置EAIP数据文件路径
- 配置大模型API Key
- 配置模型名称（GLM-5.1）
- 配置是否启用大模型
- 配置降级策略

**验收标准**：
- 支持通过配置文件修改参数
- 配置变更无需重启服务
- 配置项完整，覆盖所有需求
- 配置文件格式清晰易懂
